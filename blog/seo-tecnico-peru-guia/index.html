SEO tecnico: Core Web Vitals, schema y auditoria

Por Andres Sanchez Botta Marzo 2026 9 min de lectura

Tu web puede tener el mejor contenido del mercado peruano, pero si Google no puede rastrearla correctamente, si tarda 6 segundos en cargar o si no entiende la estructura de tus paginas, ese contenido no va a posicionar. El SEO tecnico es la base invisible sobre la que se construye todo lo demas: sin cimientos solidos, el edificio no se sostiene.

En Peru, la mayoria de webs de negocios tienen problemas tecnicos que las penalizan en Google sin que sus dueños lo sepan. Paginas que no cargan en movil, imagenes sin comprimir de 3 MB, cero datos estructurados, URLs duplicadas sin canonicals. Esta guia explica que es el SEO tecnico, que metricas mide Google, y como auditar y corregir los problemas mas comunes.

Que es el SEO tecnico y por que Google lo prioriza

El SEO tecnico se ocupa de la infraestructura de tu web: todo lo que afecta como Google rastrea, indexa y evalua tus paginas. No tiene que ver con las palabras que escribes (eso es SEO de contenido), sino con como esta construida tu web a nivel de codigo y servidor.

Google lo prioriza por una razon simple: antes de evaluar si tu contenido es relevante, su bot (Googlebot) necesita poder acceder a tus paginas, entender su estructura y cargarlas rapido. Si falla en cualquiera de esos pasos, tu contenido nunca llega a competir en los resultados de busqueda.

Los pilares del SEO tecnico son:

  • Rastreabilidad: que Googlebot pueda acceder a todas las paginas importantes de tu web sin bloqueos ni errores
  • Indexabilidad: que las paginas rastreadas se indexen correctamente y no haya duplicados ni contenido fino que diluya la autoridad
  • Rendimiento: que las paginas carguen rapido y ofrezcan una experiencia de usuario solida, medida por Core Web Vitals
  • Estructura: que Google entienda la jerarquia, el contenido y las relaciones entre paginas mediante datos estructurados, sitemaps y canonicals

Core Web Vitals: las 3 metricas que Google mide en tu web

Desde 2021, Google utiliza Core Web Vitals como factor de ranking. Son tres metricas que miden la experiencia real del usuario al cargar e interactuar con tu web. Google las documenta en su Search Central (developers.google.com/search) y las mide con datos reales de usuarios de Chrome (CrUX report).

  • LCP (Largest Contentful Paint): mide cuanto tarda en renderizarse el elemento visual mas grande de la pagina (imagen principal, bloque de texto, video). El umbral optimo es menos de 2.5 segundos. Si supera los 4 segundos, Google lo marca como "pobre". Un LCP lento suele deberse a imagenes sin comprimir, servidor lento o CSS/JS que bloquea el renderizado
  • INP (Interaction to Next Paint): reemplazo de FID desde marzo 2024. Mide la latencia de todas las interacciones del usuario (clics, toques, teclado) durante toda la visita, y reporta la peor. El umbral optimo es menos de 200 milisegundos. Un INP alto indica JavaScript pesado que bloquea el hilo principal del navegador
  • CLS (Cumulative Layout Shift): mide la estabilidad visual. Cuando una pagina se mueve mientras carga (el texto salta porque aparece una imagen arriba, un banner empuja el contenido), eso genera frustracion. El umbral optimo es menos de 0.1. Las causas mas comunes: imagenes sin dimensiones width/height, fuentes web que cargan tarde, contenido inyectado dinamicamente sin espacio reservado

Puedes medir tus Core Web Vitals gratuitamente en PageSpeed Insights (pagespeed.web.dev), que muestra tanto datos de laboratorio como datos reales de usuarios. Google Search Console tambien reporta el estado de Core Web Vitals de todo tu sitio en la seccion "Experiencia de pagina".

Auditoria SEO tecnica: que se analiza y que herramientas usar

Una auditoria SEO tecnica es una revision sistematica de todos los factores tecnicos que afectan el posicionamiento. Las areas que se analizan:

  • Rastreo e indexacion: paginas bloqueadas por robots.txt, errores 404, paginas huerfanas (sin enlaces internos), cadenas de redireccion, paginas con meta noindex accidental
  • Rendimiento: Core Web Vitals de todas las paginas, tiempo de respuesta del servidor (TTFB), peso total de la pagina, recursos que bloquean el renderizado
  • Arquitectura: profundidad de clics (ninguna pagina importante deberia estar a mas de 3 clics de la home), estructura de URLs, breadcrumbs, enlazado interno
  • Datos estructurados: implementacion de schema markup, errores en JSON-LD, tipos de schema relevantes para el negocio
  • Seguridad: certificado SSL activo, contenido mixto (recursos HTTP en pagina HTTPS), headers de seguridad
  • Mobile: responsive correcto, tamaño de fuentes legibles, elementos tactiles con espaciado adecuado

Herramientas gratuitas de Google

  • Google Search Console: muestra errores de rastreo, paginas indexadas vs excluidas, Core Web Vitals reales, problemas de usabilidad movil y datos estructurados con errores. Es la fuente mas confiable porque usa datos directos de Google
  • PageSpeed Insights: analiza cualquier URL y muestra metricas de rendimiento con recomendaciones especificas de mejora
  • Lighthouse: integrado en Chrome DevTools (F12 → pestaña Lighthouse). Audita rendimiento, accesibilidad, buenas practicas y SEO con puntuaciones de 0 a 100
  • Rich Results Test: valida si tus datos estructurados son correctos y elegibles para rich snippets en Google

Para auditorias mas completas, herramientas como Screaming Frog (version gratuita hasta 500 URLs) permiten rastrear tu sitio completo simulando lo que hace Googlebot, identificando errores sistematicos que no se ven pagina por pagina.

Schema markup: como aparecer con rich snippets en Google

Los datos estructurados (schema markup) son codigo JSON-LD que le dice a Google exactamente que contiene tu pagina: es un producto con precio, un articulo con autor, un negocio con horario, un FAQ con preguntas y respuestas. Sin este marcado, Google tiene que inferir — con el, entiende con precision.

El beneficio directo: rich snippets. En vez de aparecer con solo titulo y descripcion en los resultados de busqueda, tu resultado puede mostrar estrellas de valoracion, precios, preguntas frecuentes desplegables, horarios de negocio o enlaces de navegacion. Los rich snippets aumentan el CTR (tasa de clics) significativamente porque tu resultado ocupa mas espacio visual y transmite mas informacion.

Tipos de schema mas relevantes para negocios en Peru:

  • LocalBusiness: para cualquier negocio con ubicacion fisica. Incluye direccion, telefono, horarios, coordenadas. Subtipos: Restaurant, LegalService, MedicalBusiness, RealEstateAgent, LodgingBusiness
  • FAQPage: para paginas con preguntas frecuentes. Google puede mostrar las preguntas desplegables directamente en los resultados de busqueda
  • Product: para paginas de productos con precio, disponibilidad y valoraciones. Esencial para tiendas online
  • Article: para articulos de blog. Incluye autor, fecha de publicacion, imagen. Ayuda a Google a entender la autoria bajo criterios E-E-A-T
  • BreadcrumbList: para la navegacion de migas de pan. Mejora la visualizacion de la jerarquia de tu web en los resultados
  • Service: para describir servicios profesionales con area de cobertura y descripcion detallada

La implementacion se hace en formato JSON-LD (JavaScript Object Notation for Linked Data) dentro de un tag <script type="application/ld+json"> en el HTML. Es el formato que Google recomienda oficialmente porque no interfiere con el contenido visible de la pagina.

Arquitectura web: sitemaps, canonicals y crawl budget

La arquitectura tecnica determina como Google descubre y prioriza tus paginas:

  • Sitemap XML: un archivo (sitemap.xml) que lista todas las paginas que quieres que Google indexe, con fecha de ultima modificacion y prioridad relativa. No garantiza indexacion, pero acelera el descubrimiento de paginas nuevas y actualizadas. Google Search Console te permite enviar tu sitemap directamente
  • Canonicals: la etiqueta <link rel="canonical"> le dice a Google cual es la version principal de una pagina cuando existen versiones similares o duplicadas. Ejemplo: si /productos?color=rojo y /productos?color=azul muestran contenido casi identico, el canonical apunta a /productos. Sin canonicals, Google puede indexar duplicados y diluir la autoridad de la pagina principal
  • Crawl budget: Google asigna un presupuesto de rastreo a cada sitio — cantidad de paginas que su bot visitara en un periodo. Para sitios pequeños (menos de 500 paginas) esto rara vez es un problema. Para sitios grandes, optimizar el crawl budget (eliminando paginas de bajo valor, evitando cadenas de redirecciones, bloqueando paginas innecesarias en robots.txt) asegura que Google dedique su presupuesto a las paginas que importan
  • Robots.txt: archivo en la raiz de tu web que indica a los bots que pueden y que no pueden rastrear. Un robots.txt mal configurado puede bloquear paginas importantes o permitir el rastreo de paginas que no quieres indexar (paneles de administracion, paginas de busqueda interna, filtros duplicados)

Velocidad de carga: tecnicas reales de optimizacion

La velocidad de carga es el factor tecnico con mayor impacto directo en conversiones y posicionamiento. Las tecnicas que producen mejoras reales:

  • Optimizacion de imagenes: convertir a formato WebP (30-50% mas liviano que JPEG/PNG sin perdida perceptible de calidad), comprimir con herramientas como Squoosh o Sharp, definir width y height en el HTML para evitar CLS, implementar lazy loading para imagenes below the fold
  • Minificacion de CSS y JavaScript: eliminar espacios, comentarios y codigo no utilizado. Herramientas como CSSO para CSS y Terser para JavaScript reducen el peso de los archivos un 20-40%
  • Eliminacion de recursos que bloquean el renderizado: CSS critico inline (el CSS minimo necesario para renderizar above the fold), carga asincrona de fuentes web con font-display: swap, atributo defer en scripts no criticos
  • Optimizacion del servidor: TTFB (Time to First Byte) menor a 200ms, compresion GZIP o Brotli habilitada, headers de cache con tiempos largos para recursos estaticos, HTTP/2 o HTTP/3 activo
  • Prefetch y preconnect: <link rel="preconnect"> para dominios externos criticos (Google Fonts, CDNs), <link rel="preload"> para recursos criticos como fuentes y CSS

Un sitio bien optimizado en Peru deberia cargar en menos de 2 segundos en una conexion 4G. La diferencia entre una web que carga en 1.5 segundos y una que carga en 5 segundos puede ser del 50% en tasa de rebote — cada segundo extra de espera pierde visitantes que no regresan.

SEO tecnico vs SEO de contenido: cual necesitas primero

Es la pregunta mas frecuente: ¿invierto primero en arreglar la parte tecnica o en crear contenido? La respuesta depende del estado actual de tu web:

  • Si tu web tiene problemas tecnicos graves (no carga en movil, errores 404 masivos, sin HTTPS, paginas bloqueadas en robots.txt), el SEO tecnico va primero. No tiene sentido crear contenido si Google no puede rastrearlo
  • Si tu web funciona razonablemente bien (carga en menos de 3 segundos, es responsive, tiene HTTPS) pero no tiene contenido, el SEO de contenido va primero. Puedes optimizar la parte tecnica en paralelo, pero el contenido es lo que genera trafico
  • Si ambos estan mal, la secuencia es: corregir los bloqueos tecnicos criticos (HTTPS, errores de rastreo, mobile) → crear contenido optimizado → hacer una auditoria tecnica completa para afinar

La analogia mas clara: el SEO tecnico es la carretera y el SEO de contenido es el vehiculo. Necesitas ambos, pero si la carretera tiene huecos enormes, da igual que tan bueno sea tu auto. Y si la carretera es perfecta pero no tienes vehiculo, no llegas a ningun lado.

Cuanto cuesta una auditoria SEO tecnica en Peru

Los rangos del mercado segun el alcance:

  • Auditoria basica (web pequeña, hasta 20 paginas): S/800-1,500. Incluye: analisis de Core Web Vitals, revision de indexacion en Search Console, verificacion de datos estructurados, recomendaciones de velocidad y un informe con acciones priorizadas
  • Auditoria completa (web mediana, 20-100 paginas): S/1,500-2,500. Agrega: rastreo completo con Screaming Frog, analisis de arquitectura y enlazado interno, revision de canonicals y duplicados, analisis de log files del servidor, plan de implementacion detallado
  • Auditoria avanzada (e-commerce o sitio grande, 100+ paginas): S/2,500-4,000+. Agrega: analisis de crawl budget, auditoria de JavaScript rendering (para SPAs y sitios con renderizado dinamico), benchmarking competitivo, setup de monitoreo continuo

La auditoria es el diagnostico — la implementacion de las correcciones es un costo aparte. Muchos desarrolladores ofrecen paquetes que incluyen auditoria + implementacion. El retorno de inversion es medible: las mejoras tecnicas generan incrementos de trafico organico que se pueden rastrear directamente en Google Analytics y Search Console.

Preguntas frecuentes

El SEO tecnico se enfoca en la infraestructura de tu web: velocidad de carga, datos estructurados, arquitectura de URLs, sitemap, canonicals y Core Web Vitals. El SEO de contenido se enfoca en las palabras clave, la calidad de los textos y la relevancia tematica. Ambos son necesarios, pero sin una base tecnica solida, el mejor contenido no posiciona.

Una auditoria SEO tecnica profesional en Peru cuesta entre S/800 y S/3,000 dependiendo del tamaño del sitio. Para webs pequeñas (hasta 20 paginas) el rango es S/800-1,500, para webs medianas entre S/1,500-2,500, y para e-commerce o sitios grandes puede superar los S/3,000.

Las señales mas comunes: tu web tarda mas de 3 segundos en cargar, Google Search Console muestra errores de rastreo o paginas no indexadas, PageSpeed Insights muestra puntuaciones por debajo de 50, o tus Core Web Vitals estan en rojo. Puedes hacer una revision inicial gratuita con PageSpeed Insights y Google Search Console.

Se recomienda una auditoria tecnica completa cada 6-12 meses, o inmediatamente despues de cambios importantes como rediseño web, migracion de dominio o cambio de hosting. El monitoreo continuo de Core Web Vitals y errores de rastreo en Google Search Console deberia ser mensual.
